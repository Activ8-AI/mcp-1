## ğŸ› ï¸ Preparing Great Support Requests

To get the best help quickly, include:

### For Bugs
```markdown
**What happened?** Clear description of the issue
**Expected behavior?** What should have happened
**How to reproduce?** Step-by-step instructions
**Environment:**
- Go version: `go version`
- Server version: `v1.0.0` or commit hash
- Operating system: macOS/Linux/Windows
- Authentication method: Bearer token/OAuth2

**Code sample:** Minimal reproduction case
**Error message:** Full error output
```

### For Questions
```markdown
**What are you trying to do?** Your goal or use case
**What have you tried?** Previous attempts and results
**Environment:** Go version, server version, OS
**Code context:** Relevant code snippets (sanitized)
**Specific question:** What exactly you need help with
```

### For Feature Requests
```markdown
**What problem does this solve?** The use case
**Proposed solution:** How you envision it working
**Alternative approaches:** Other ways you've considered
**Priority:** How important is this for your project
```
